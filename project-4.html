United Way Stair Climb Website (November 2024)

Developed a full-stack website to support event registration and participant tracking for United Way’s Stair Climb.

Problem/Goal

United Way needed a centralized, accessible platform to allow participants to register and track their steps, while also providing event managers with tools to oversee and update participant data. Key requirements included a live leaderboard, step entry forms, and an admin panel to edit, delete, or manually add participant records.

My Role

As a backend developer on the TSI team, I helped build and maintain core functionality for the website. This included setting up API routes, managing CSV data handling, and ensuring smooth communication between the frontend (React) and backend (Flask).

Process

Planning
We kicked off the project by meeting with United Way to gather detailed functional requirements. After weighing options like a mobile app versus a web app, we agreed on a responsive website for broader accessibility. Our team then divided responsibilities between frontend and backend and established a collaborative GitHub workflow.

Development
Using Flask for the backend and React for the frontend, we developed a simple and intuitive web interface. I focused on setting up server-side logic for data handling, including CSV file management and API endpoints to support registration and step tracking.

Testing
Features were tested iteratively as they were built. Once a feature passed local testing and peer review, it was merged into the main GitHub branch. This approach helped us maintain a stable build throughout development.

Collaboration
Frequent team meetings kept everyone aligned on progress and deadlines. We used GitHub Issues and group chats to track tasks and troubleshoot in real-time, which was crucial given our tight timeline and varying schedules.

Challenges and Solutions

The biggest challenge was the accelerated timeline, United Way’s event started in October, leaving us just weeks to deliver a working product. Many of us, myself included, had limited experience with React and Flask, which meant learning on the fly. I adapted quickly, teaching myself the necessary frameworks and contributing production-ready code under pressure.

Outcome

Despite the time constraints and steep learning curve, our team successfully delivered a working platform on time. The website was used by participants and event managers throughout the Stair Climb, and United Way was pleased with the functionality and user experience.
