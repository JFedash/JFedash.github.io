United Way Stair Climb Website (November 2024)

Developed a full-stack website for participant registration, step tracking, and real-time leaderboard functionality in support of United Way’s annual Stair Climb event.

Overview

As part of a student-led team in Tech for Social Impact (TSI), I contributed to the development of a full-stack web platform for United Way’s Stair Climb Event. Built using React and Flask under tight deadlines, the platform enabled participants to register, log steps, and view a live leaderboard. It also featured an admin dashboard for event managers to manually update participant records. The website was successfully deployed and used during the event, highlighting my adaptability and backend development capabilities.

Problem/Goal

United Way needed a centralized, user-friendly platform to streamline participant registration and step tracking. The system also required administrative tools for managing data, such as editing, deleting, and manually entering participant information. Our goal was to create a responsive, reliable web app to meet these needs before the event’s launch.

My Role

As a backend developer, I focused on building Flask API routes and managing data logic. I was responsible for handling CSV file imports, step tracking, and ensuring reliable communication between the React frontend and Flask backend.

Process

Planning
We began by gathering requirements from United Way and evaluating options for implementation. After deciding on a web app for its broader accessibility, we divided responsibilities between frontend and backend and established a GitHub workflow to keep development organized.

Development
The site was built using React for the frontend and Flask for the backend. I contributed to the backend by supporting data management, building API routes, and integrating CSV functionality to power registration, step tracking, and admin tools.

Testing
Each feature underwent peer review and local testing before being merged into the main GitHub branch. This incremental approach helped us avoid major bugs and maintain a stable build as we neared deployment.

Collaboration
Our team coordinated through regular meetings, GitHub Issues, and messaging channels. We worked efficiently despite time constraints and varying schedules, using version control and task-tracking tools to stay aligned.

Challenges and Solutions

One of the main challenges was learning unfamiliar technologies while actively building the platform. I had limited prior experience with Flask, so I took the initiative to teach myself the framework while contributing to backend development. As we encountered issues integrating the frontend and backend, I collaborated closely with the team to troubleshoot and ensure consistent data flow. Balancing development with other academic responsibilities also required strong time management, which I maintained by breaking tasks into smaller components and contributing steadily over time.

Outcome

We delivered a fully functional platform on time, which was used successfully by both participants and event staff throughout the Stair Climb. United Way was pleased with the result, and the project demonstrated my ability to quickly learn new tools, collaborate effectively, and deliver under pressure.
