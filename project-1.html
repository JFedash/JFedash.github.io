DataQuest AI Hackathon (April 2024)

Built a machine learning model to detect fraudulent transactions using real-world banking data during a high-pressure three-day hackathon.

Overview

During a three-day hackathon, I led the development of a fraud detection system using real-world banking data. With no prior machine learning experience, our team implemented a random forest classifier in Python, emphasizing rapid prototyping and continuous iteration. Despite the tight timeline, we delivered a functional model that successfully identified approximately 75 percent of fraudulent transactions.

Problem/Goal

The challenge was to classify banking transactions as fraudulent or legitimate using a structured dataset containing features like location, time, and amount. Our task was to design a machine learning model that could accurately flag fraudulent activity in an unseen test set.

My Role

As the technical lead, I oversaw the machine learning pipeline from start to finish. This included researching model options, handling data preprocessing and feature selection, training and tuning the model, and evaluating its performance. While a teammate supported implementation, I took primary responsibility for the core technical components, while two others focused on presentation and documentation.

Process

Planning
With the challenge topic revealed on day one, I began by researching common ML models and evaluation techniques. We adopted a flexible, prototype-driven approach to allow for rapid iteration and learning.

Development
We used Python to build a random forest classifier. I led the implementation of the preprocessing pipeline, tuned model parameters to reduce overfitting, and continuously refined our feature set to improve accuracy.

Testing
Evaluation was central to our workflow. I set up repeatable testing using precision, recall, F1-score, and accuracy to measure and improve performance as we iterated.

Collaboration
I worked closely with my co-developer to problem-solve and exchange ideas while keeping our team aligned on progress. We regularly shared updates to help the presentation team effectively communicate our methods and results.

Challenges and Solutions

With no prior machine learning experience, I had to quickly learn the fundamentals while leading the technical direction. Time constraints demanded fast decision-making and experimentation. By focusing on simplicity and adaptability, we avoided scope creep and kept our solution aligned with the goal.
